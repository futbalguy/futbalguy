Server Side Swift with Vapor

2016-08-14

I recently took the server-side Swift framework Vapor for a spin and want to share the experience!

Swift has always been touted as a multipurpose language, but since it was open-sourced developers have been accelerating the rate of awesome projects. Vapor seems to have an especially energetic and inviting group of contributors. I even contributed a few PRs as I found bugs It has been a really great learning experience to participate in the development as it reached version 1.0 alongside the release of Swift 3.0.

As a disclaimer, much of my prior backend experience was with Node.js and the Express framework so that has informed by perspective. I was hoping Vapor could be as easy to understand as Express, but with the added protection from programmer error that is a big part of Swift’s focus (i.e. strong-typing, optionals, etc.).
